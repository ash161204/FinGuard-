Execution Plan

This plan assumes:

single demo user
FastAPI BackgroundTasks for v1 jobs
local-disk uploads
separate Node HTTP engine service
manual correction for extracted Form 16 and CAMS data before analysis
Milestone 0: Architecture Freeze
Scope: lock contracts, folder structure, service boundaries, and v1 non-goals.
Deliverables: shared API spec, canonical JSON schemas, repo layout, env var list, sequence diagram, decision log.
Exit criteria: endpoints, job states, engine request/response shape, Form 16 tier rules, and score/FIRE input contracts are frozen.

Milestone 1: Repo Scaffolding
Scope: create the project skeleton for frontend, backend, engine-service, and docs.
Deliverables: Expo app booting, FastAPI app booting, Node service booting, lint/test/prettier setup, base README, .env.example.
Exit criteria: all three services run locally and basic health endpoints respond.

Milestone 2: Engine Service Integration
Scope: wrap insightEngine.js behind a stable Node microservice.
Deliverables: POST /run-engine, typed request modes for tax, mf, score, input validation, error handling, fixture tests.
Exit criteria: backend can call the Node service and receive deterministic JSON for sample tax and MF payloads.

Milestone 3: Engine Contract Patching
Scope: patch engine outputs so downstream services do not need brittle hacks.
Deliverables: canonical tax output fields for scoring, canonical MF allocation fields, overlap field alignment, monthly-income naming fixes, removal of bank-analysis exposure.
Exit criteria: score service can consume tax and MF outputs without special-case field remapping.

Milestone 4: Backend Core and Database
Scope: build the FastAPI foundation and persistence layer.
Deliverables: SQLAlchemy models, Alembic migrations, repositories, demo-user bootstrap, tables for profiles, extracted data, normalized data, derived data, actions, score history, and jobs.
Exit criteria: database migrates cleanly and CRUD flows work for the demo user.

Milestone 5: Upload and Job System
Scope: support PDF upload, local storage, job creation, and polling.
Deliverables: /upload/form16, /upload/cams, GET /job/{job_id}, job state transitions, upload validation, file path management, structured job errors.
Exit criteria: uploaded files are persisted locally, jobs progress through pending -> processing -> completed/failed, and the frontend can poll status.

Milestone 6: Extraction Pipeline
Scope: implement text extraction, OCR fallback, Gemini structuring, and validation.
Deliverables: PDF text extractor, Tesseract OCR path, cleaned text preprocessing, regex helpers, Gemini function-calling client, PII stripping, strict schema validation, retry-once logic.
Exit criteria: Form 16 and CAMS sample PDFs produce validated extracted JSON with null for unknown values and a missing-fields list.

Milestone 7: Tiered Form 16 and CAMS Review Flow
Scope: enforce the tier model and support user correction before analysis.
Deliverables: Form 16 Level 1/2/3 schema handling, critical-field blocker logic, editable extraction payload API, persistence of reviewed values, CAMS correction support.
Exit criteria: Tax Wizard blocks only when Level 1 is missing, optional fields do not block, and corrected values can be saved and reused.

Milestone 8: Normalization Layer
Scope: convert extracted/reviewed payloads into engine-ready canonical JSON.
Deliverables: field-map config, number/date normalization, source merge rules, null-safe transforms, normalization tests, persisted normalized profile records.
Exit criteria: both Form 16 and CAMS reviewed inputs normalize consistently into engine-compatible payloads.

Milestone 9: Analysis APIs
Scope: implement deterministic backend flows for Tax and MF analysis.
Deliverables: /analyze/tax, /analyze/mf, backend adapters for engine output, top-insight ranking, UI-friendly insight transformation, persistence in derived_data.
Exit criteria: both endpoints return summary, top_insights, and full_report from fixture inputs, including partial-analysis behavior.

Milestone 10: Score and FIRE APIs
Scope: implement Money Health Score and FIRE planner.
Deliverables: /score, /fire, monthly-income-based score input handling, FIRE formulas, yearly plan generation, corpus estimation, score history persistence.
Exit criteria: score and FIRE return stable outputs from known fixtures, and score changes are written to history.

Milestone 11: Action Tracking
Scope: build inferred and manual action management.
Deliverables: action creation from ranked insights, status/progress updates, top-3 home feed, backend persistence, action APIs.
Exit criteria: Home can show top actions and users can update progress/status.

Milestone 12: Frontend App Shell
Scope: implement navigation, state, API integration, and app-wide infra.
Deliverables: stack/tab navigation, Zustand store, React Query client, AsyncStorage cache, API client, loading/error primitives, theme tokens.
Exit criteria: the app navigates end-to-end and loads mock/real backend data.

Milestone 13: Upload and Review UX
Scope: build the upload, polling, review, and retry flows.
Deliverables: Upload screen, job polling UI, extraction review forms, missing-field prompts, retry states, blocked-state messaging, partial-analysis messaging.
Exit criteria: user can upload, wait for extraction, correct fields, and trigger analysis without leaving the app.

Milestone 14: Feature Screens
Scope: build the user-facing Tax, MF, FIRE, Home, and Profile experiences.
Deliverables: Home score card, trend graph, top 3 actions, Tax summary/detail screens, MF summary/detail screens, FIRE planner screen, profile editor.
Exit criteria: all main screens render real data and support empty, partial, and success states.

Milestone 15: Caching and Data Invalidation
Scope: make the app resilient and consistent.
Deliverables: React Query cache keys, invalidation on edits/uploads/profile changes, AsyncStorage hydration, optimistic UI only where safe.
Exit criteria: changing source data invalidates dependent reports and stale outputs are not shown.

Milestone 16: Testing and Fixtures
Scope: harden the system before deployment.
Deliverables: unit tests for normalization/ranking/FIRE math, engine adapter tests, backend integration tests, sample Form 16 and CAMS fixtures, frontend smoke tests.
Exit criteria: core paths are covered and fixture outputs are reproducible.

Milestone 17: Documentation and Deployment
Scope: finish developer docs and ship v1 to Render.
Deliverables: full README, architecture diagram, setup guide, API docs, limitations section, Render configs for backend and engine service, env setup instructions.
Exit criteria: both services deploy successfully to Render, environment configuration is documented, and the v1 demo flow works in deployed form.
